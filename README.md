# Cognifyz Full Stack Internship Projects

## Overview

This repository contains my Cognifyz Full Stack Internship task submissions. I have organized each task in its own folder so every concept is easy to open, run, test, and review.

The projects move step by step from basic Express and EJS pages to form validation, responsive design, client-side JavaScript, API integration, and MongoDB authentication.

## Tasks Included

| Task | Project | Focus |
| --- | --- | --- |
| 1 | `Task-1-HTML-Server` | Basic Express server setup, EJS rendering, and form submission |
| 2 | `Task-2-Form-Validation` | Form validation, server-side checks, error messages, and submitted data display |
| 3 | `Task-3-Responsive-Design` | Responsive page layout, improved styling, and mobile-friendly design |
| 4 | `Task-4-DOM-Validation` | Client-side validation, DOM events, and interactive form behavior |
| 5 | `Task-5-API-Integration` | REST API routes with frontend Fetch API interaction |
| 6 | `Task-6-Database-Authentication` | MongoDB authentication with registration, login, JWT cookies, and protected dashboard |

## How to Run Any Task

1. Open the task folder you want to test:
   ```bash
   cd Task-5-API-Integration
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Configure environment variables if that task includes a `.env.example` file.
4. Start the server:
   ```bash
   npm start
   ```
5. Open the local URL mentioned in that task's README file.

## Task Notes

- Tasks 1 to 4 focus on Express, EJS, styling, validation, and frontend interactivity.
- Task 5 demonstrates API routes and browser-side `fetch` requests.
- Task 6 adds MongoDB, Mongoose models, password hashing, JWT cookies, and route protection.
- Each folder has its own README with setup steps and task-specific details.

## Important Notes

- Do not upload `node_modules` in the final submission.
- Keep `.env` files private and use `.env.example` for sample environment variables.
- Database-based tasks require MongoDB to be running locally or a valid MongoDB connection string.
- Screenshots or demo videos are stored inside the task folders when required.
